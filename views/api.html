API Endpoints

GET: /api/topics

Returns a JSON object containing all the current topics.

GET: /api/topics/ topic_id/articles

Returns a JSON object containing all the articles for the given topic_id.

GET: /api/articles

Returns a JSON object containing all the current articles.

GET: /api/articles article_id

Returns a JSON object containing a specific article identified by the given article_id.

PUT: /api/articles/ article_id

Increment or Decrement the votes of an article by one. The article is identified by the given article_id. This route requires a query of ?vote=up or ?vote=down.
e.g. /api/articles/1234567890?vote=down

GET: /api/articles/ article_id /comments

Returns a JSON object containing all the comments for the given article_id.

POST: /api/articles/ article_id /comments

Add a new comment to an article using the article_id to identify the article. This route requires a JSON body with a comment key and String value as it's pair.
body: { "comment": "This is my new comment" }

GET: /api/comments/ comments_id

Returns a JSON object containing a single comment identified by the given comments_id.

PUT: /api/comments/ comments_id

Increment or Decrement the votes on a comment by one. The article is identified by the given comment_id. This route requires a query of ?vote=up or ?vote=down.
e.g. /api/comments/1234567890?vote=up

DELETE: /api/comments/ comments_id

Delete the comment identified by the given comment_id.

GET: /api/users/ username

Returns a JSON Object containing the profile data for the user with the given username.
